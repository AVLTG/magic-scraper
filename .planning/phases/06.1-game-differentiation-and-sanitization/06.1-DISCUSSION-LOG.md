## Import marker + backfill

### Q1: How should we mark the ~20 legacy games vs new live games at the data layer?

| Option | Description | Selected |
|--------|-------------|----------|
| Boolean isImported on Game | Add `isImported Boolean @default(false)` to Game model | ✓ |
| String source enum | `source String @default("live")` with zod narrowing | |
| Notes substring match | No schema change; `WHERE notes NOT LIKE '%Ported from Spreadsheet%'` | |
| New GameSource tag table | Separate table joined to Game | |

**User's choice:** Boolean isImported on Game (Recommended).

---

### Q2: How should we backfill the ~20 existing production games with this marker?

| Option | Description | Selected |
|--------|-------------|----------|

**User's choice:** One-off SQL in a plan step (primary), with willingness to fall back to manual UI edit if needed ("I am fine with the one-off sql in a plan step but if needed I dont mind doing the manual UI edit either. 20 is not that crazy").

**Notes:** Claude treats SQL as the primary path in CONTEXT.md D-03; manual UI path is documented as a contingency fallback.

### Q3: Should the import marker be editable via the GameForm UI after creation?

| Option | Description | Selected |
|--------|-------------|----------|
| No — hidden, set only by backfill / API | Flag never shown in UI; only SQL can change it | ✓ |
| Yes — visible checkbox, admin-only edit | Toggle on edit page only, for misclassification fixes | (conditional) |

**User's choice:** Conditional — "Is edit currently admin-only? If so, go with 2. If not, keep it 1."

**Notes:** Claude verified that edit is NOT admin-only (regular shared-password users can edit any game via `/games/[id]/edit`), so Option 1 wins. User also added a critical semantic clarification: "All imported data is counted as missing 'Won by combo' and 'Deck Name' fields, even if they are checked or have a name listed." — this shaped D-06 in CONTEXT.md.

### Q4: When we add the new column, should we also remove 'Ported from Spreadsheet' from the notes field of those 20 games?

| Option | Description | Selected |
|--------|-------------|----------|
| Leave notes untouched | Redundant indicators; notes stay as human-readable marker | ✓ |
| Strip the marker substring | Clean up notes in the same backfill SQL | |
| Replace notes entirely with NULL | Risks losing other text alongside the marker | |

**User's choice:** Leave notes untouched. User clarified: "if editability is not going to be visible, will the new column? Dont feel its necessary, meaning ported from spreadsheet note should be the only thing people see, it should not show up in the list of games."

**Interpretation:** The `isImported` column and any badge/indicator for it should NOT appear in the UI. The notes text is the exclusive visible indicator. The column is backend-only, used only by Phase 7 stat queries and as a row-creation default.
