# Phase 06.1 Plan 02: Duplicate Player Prevention Summary

**One-liner:** Prevent duplicate participants in a single game via a three-layer defense — client-side Combobox `excludeItems` filter, D-12 disabled collision row, and zod `.refine()` enforcing case-insensitive uniqueness on `gameSchema.participants`.

## Objective Recap

Close the Phase 6 sanitization gap (GAME-09) where `GameForm` happily accepted two "Amirali" entries, corrupting future Phase 7 per-player stats. Three defenses stacked:

1. **Client UX filter (D-10, D-11, D-13):** Combobox gains `excludeItems?: string[]` prop; GameForm derives per-row exclusion list from other filled rows and passes it in. Live-computed on every render.
2. **Client UX collision notice (D-12):** Typing a colliding name swaps the `Add "xyz" as new player` affordance for a disabled, non-clickable `Player already in game` row. ArrowUp/ArrowDown/Enter skip it.
3. **Server defense-in-depth (D-14, Phase 6 D-29):** `gameSchema.participants.refine(...)` rejects case-insensitive duplicates with the literal message `duplicate player names not allowed`. Both POST `/api/games` and PATCH `/api/games/[id]` already call `gameSchema.parse(...)` — zero route changes.

## Tasks Completed

### Task 1 — zod refine for duplicate player names (commit `002129e`)

**Files:** `src/lib/validators.ts`, `tests/games-api.test.ts`

Added `.refine((arr) => new Set(arr.map((p) => p.playerName.toLowerCase())).size === arr.length, { message: "duplicate player names not allowed" })` on `gameSchema.participants`. Inline D-14 rationale comment explains why the refine lives there (defense-in-depth) and why case-insensitive comparison is safe (because `gameParticipantSchema.playerName.trim()` normalizes whitespace first).

6 new unit tests in `tests/games-api.test.ts` (new `describe('gameSchema duplicate-participant refine (D-14)', ...)` block placed at top of file, after imports):
- accepts distinct player names
- rejects identical duplicate player names
- rejects case-insensitive duplicate player names (`'Amirali'` vs `'amirali'`)
- treats whitespace-trimmed names as duplicates (`'Amirali'` vs `'  Amirali  '`)
- accepts single participant
- rejects 4 participants with one duplicate (`['A', 'B', 'C', 'a']`)

**Zero route changes:** confirmed by `git diff --name-only src/app/api/games/route.ts src/app/api/games/[id]/route.ts` returning empty — routes already call `.parse()` so the refine enforces automatically per Phase 6 D-29.

**Render pipeline changes (inside `Combobox` function body):**

**Listbox render:** new `{showExcluded && (...)}` branch renders a `<li role="option" aria-disabled="true" aria-selected={false}>` with `cursor-not-allowed select-none opacity-60` classes, no `onMouseDown` handler, displaying the `excludeLabel` text. ### Task 3 — excludeItemsForRow helper wired into GameForm (commit `6d642eb`)

**Files:** `src/app/games/game-form.tsx`, `tests/game-form.test.ts`

**New exported pure helper:**

```typescript
export function excludeItemsForRow(
  rowIndex: number,
  state: { rows: ParticipantRow[] }
): string[] {
  return state.rows
    .map((r, i) => ({ name: r.playerName.trim(), i }))
    .filter((r) => r.i !== rowIndex && r.name.length > 0)
    .map((r) => r.name);
}
```

Signature takes the whole state object (not just `rows`) so future additions — e.g. a winnerIndex-aware filter — don't need a signature change.

**Wiring:** The per-row player-name `<Combobox>` inside the `state.rows.map((r, i) => ...)` block now passes:

```tsx
<Combobox
  items={playerItems}
  value={r.playerName}
  onChange={(v) => updateRow(i, { playerName: v })}
  placeholder={`Player ${i + 1}`}
  addLabel="player"
  excludeItems={excludeItemsForRow(i, state)}
  excludeLabel="Player already in game"
/>
```

**Deck Combobox intentionally unchanged** — duplicate deck names are valid (two players can share a deck; same deck across games is normal). D-10 scope is explicitly player names only.

**No memoization:** O(4) per-row derivation is cheaper than `useMemo` bookkeeping; consistent with D-13 live-update semantics.

8 new unit tests in a new `describe('excludeItemsForRow (D-10, D-11)', ...)` block:
- returns all other filled rows for rows 0, 1, 2, 3
- returns empty array when all rows empty
- treats whitespace-only rows as empty
- trims leading/trailing whitespace from included names
- does NOT include the caller row's own name (D-11)
- handles 4-row full state for row 3
