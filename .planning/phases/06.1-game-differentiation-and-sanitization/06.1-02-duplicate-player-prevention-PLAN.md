<objective>
Prevent the same player from being listed twice in a single game along three defenses:

1. **Client UX filter (D-10, D-11, D-13):** Extend `Combobox` with `excludeItems?: string[]`. Wire `GameForm` to derive a per-row exclusion list from the OTHER filled rows' player names, passing it into each player-name Combobox. Derived on every render — no new state, no API calls.
2. **Client UX collision notice (D-12):** When the user types a name that collides with an already-filled row, the Combobox swaps the existing "Add 'xyz' as new player" affordance for a disabled, non-clickable row reading `"Player already in game"`. Keyboard navigation skips it.
3. **Server defense-in-depth (D-14, Phase 6 D-29):** Add a `.refine()` on `gameSchema.participants` that rejects duplicate `playerName` values case-insensitively with the message `"duplicate player names not allowed"`. Both `POST /api/games` and `PATCH /api/games/[id]` already call `gameSchema.parse(...)`, so zero route-file changes are needed.

Purpose: Closes the Phase 6 sanitization gap (GAME-09) for duplicate participants — currently the form will happily accept two "Amirali" entries, corrupting Phase 7 per-player stats.

Output: Updated Combobox with excludeItems pathway, GameForm wiring it per-row, zod refine on gameSchema, and new unit tests for both layers.
</objective>

    export function filterItems(items: string[], inputValue: string, excludeItems?: string[]): string[] {
      const q = inputValue.toLowerCase();
      const excluded = new Set((excludeItems ?? []).map((x) => x.trim().toLowerCase()).filter((x) => x.length > 0));
      return items.filter((i) => {
        if (excluded.has(i.toLowerCase())) return false;
